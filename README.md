# game-stats

An exercise to practice OOP, Junit tests, and maps.

## Wave 0: Setup and Understand
FORK and clone this repo. `cd` into the repo directory and open VS Code. MAKE SURE YOU ARE IN THE game-stats directory before opening VS Code otherwise the tests won't be detected!

Open GameStatsRunner. You will not need to modify this code, but we will run it to validate your soltuions. Run GameStatsRunner by opening it and clicking the play button in VS Code. Enter `scores.txt` as the filename when prompted. Try using it, but know that it will tell you that most of the functionality is not there yet because you have not implemented it.

Look at MapGameStatsCalculator.java. You will be implementing this file for the project. Take both a look at this file and the tests.

## Wave 1: gameCount

In this wave we will implement gameCount. To achieve this we will need to modify both the constructor and the gameCount method itself. Most of the logic will go in the constructor!

Stuck and need a hint? Check out hints/wave1.md to get a hint.

After your initial implementation, add/commit/push!

Try running the tests and see if your solution passes. We are only trying to get the tests for gameCount to pass in this wave, all other tests will still fail because you have not implemented the other methods. Fix any bugs you find.

Then, add more tests! Consider things like what if we only have one person, what if that person has games with a negativs score, etc.

Add/commit/push!
